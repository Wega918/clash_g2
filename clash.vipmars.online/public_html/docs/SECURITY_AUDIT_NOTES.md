# Что исправлено в серверной логике

## Основные правки
1. Все изменения баланса в ключевых серверных обработчиках переведены на атомарные операции.
2. Добавлена единая валидация допустимых ресурсных колонок (`gold`, `elixir`, `dark_elixir`, `gems`).
3. Для игровых действий в `storage_router.php` и `building_router.php` теперь обязателен CSRF-токен даже при GET/XHR.
4. Исправлен риск потери ресурсов в `army_api.php`: раньше при ошибке добавления в очередь ресурс уже мог быть списан.
5. Возврат ресурсов за отмену обучения переведён на централизованное серверное зачисление.

## Почему это важно
- нельзя подменить имя колонки ресурса через косвенный вызов;
- нельзя увести баланс в минус через гонку нескольких одновременных запросов;
- нельзя спровоцировать покупку/апгрейд простым GET без корректного токена;
- нельзя потерять ресурс между списанием и постановкой в очередь.

## Что ещё стоит сделать следующим патчем
- вынести все изменения ресурсов вообще в один service-layer файл;
- добавить журнал `resource_ledger` для аудита всех списаний/начислений;
- перевести все игровые действия только на POST JSON API;
- закрыть dev-ошибки на production (`display_errors=0`);
- добавить rate-limit на критические action endpoints.
